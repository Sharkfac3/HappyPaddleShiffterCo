# Arduino Mega 2560 — Overview

## Hardware Specifications

| Attribute | Value |
|---|---|
| Microcontroller | ATmega2560 (Microchip/Atmel) |
| Operating voltage | 5V |
| Input voltage (recommended) | 7–12V (via VIN or DC barrel jack) |
| Input voltage (absolute max) | 6–20V |
| Digital I/O pins | 54 (pins 0–53) |
| Analog input pins | 16 (A0–A15; also usable as digital I/O) |
| PWM outputs | 15 (pins 2–13, 44–46) |
| Max current per I/O pin | 40mA (absolute max); 20mA recommended |
| Max total current (all pins) | 200mA |
| Flash memory | 256 KB (8 KB used by bootloader) |
| SRAM | 8 KB |
| EEPROM | 4 KB |
| Clock speed | 16 MHz |
| USB interface | ATmega16U2 USB-to-serial bridge |
| Connector | USB-B (standard square printer connector) |

---

## Power Outputs

| Pin | Voltage | Max Current | Notes |
|---|---|---|---|
| 5V | 5V | 500mA (from USB) / 1A (from VIN) | Logic power for Arduino and most peripherals |
| 3.3V | 3.3V | 50mA | Limited current — suitable for the SSD1351 display (low current SPI device) |
| GND | 0V | — | Multiple GND pins available |

> ⚠️ The 3.3V pin on the Mega is generated by a small linear regulator — it is current-limited. Do not draw more than 50mA from it. The SSD1351 OLED display draws well under this limit.

---

## Key Constraints for This Project

1. **Hardware SPI pins are fixed** — see [`hardware-spi.md`](hardware-spi.md)
2. **Analog pins A0–A15 can be used as digital I/O** — the display CS/DC/RST are wired to A3, A4, A5 as digital outputs
3. **Pins 0 and 1 are used by Serial (TX/RX)** — do not use them for other purposes if Serial is in use
4. **Pins 2 and 3 support hardware interrupts** — they are used for the paddle switches; consider using hardware interrupt handlers if debounce timing becomes critical
5. **No solenoid loads on Arduino pins** — always use external driver board
