# Arduino Mega 2560 — Hardware Reference

## Why the Mega 2560

The Arduino Mega 2560 is used (not the Uno) because:
- More digital I/O pins — this project uses 13 digital pins
- Hardware SPI on dedicated pins (51, 52) — the Uno's hardware SPI is on pins 11, 13,
  which conflicts with the solenoid outputs in this project
- More memory — useful for the Adafruit GFX/SSD1351 libraries

## Hardware SPI — Critical Notes

On the Mega 2560, hardware SPI is on:
- **MOSI → Pin 51**
- **SCK → Pin 52**
- **MISO → Pin 50** (not used by this project)
- **SS → Pin 53** (not used directly; managed by CS)

These are DIFFERENT from the Uno (Uno uses 11/12/13). This is a common source of
confusion when adapting Uno code to the Mega.

The SSD1351 display library in hardware SPI mode uses these fixed pins. They cannot
be reassigned without switching to software SPI (slower, not recommended).

## Free Pins (Available for Future Use)

Digital: 8, 9, 10, 14–50, 53
Analog: A0–A2, A6–A15

Notable free pins: hardware UART ports (Serial1–3 on pins 14–19) are available
for future CAN bus, data logging, or telemetry integration.

## Memory

- Flash: 256 KB (programme storage)
- SRAM: 8 KB (runtime variables)
- EEPROM: 4 KB (non-volatile storage — not currently used)

The Adafruit SSD1351 + GFX libraries have a significant flash footprint (~25 KB).
Monitor flash usage. SRAM usage with the current four classes is low.

## Serial

Serial0 (pins 0/1 / USB) used for debug output at 9600 baud.
`Serial.println()` calls throughout the firmware are debug only — they can be
removed for production builds to reclaim a small amount of flash.
